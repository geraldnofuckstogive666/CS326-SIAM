# Support Plan – TaskLite

## Overview
This document defines how support and issue handling are managed for the TaskLite system.

---

## Support Scope
- Bug fixes
- Minor UI improvements
- Deployment issue resolution

---

## Issue Reporting Process
Users report issues via:
- GitHub Issues
- Direct documentation logs

---

## Response Time Policy
- Critical issues: 24 hours
- Medium priority: 2–3 days
- Low priority: 5 days

---

## Maintenance Approach
- Regular code review after each sprint
- Continuous monitoring via logs and health endpoint
- Incremental updates via GitHub workflow

---

## Final Notes
Support is lightweight and academic-focused, not enterprise-level.
